# Plan: Rediseñar Footer - Eliminar position fixed

## Objetivo
Cambiar el footer de `position: fixed` (siempre visible en el viewport) a un footer que fluya naturalmente después del contenido de la página.

## Archivos a modificar

### 1. `src/components/Footer.jsx`
- **Cambiar:** Eliminar las clases `fixed bottom-0 left-0 right-0 z-50` del `<footer>`
- **Reemplazar por:** clases de posicionamiento normal (flujo del documento)
- **Resultado:** El footer aparece justo después del contenido de cada página

### 2. `src/App.jsx`
- **No se requieren cambios estructurales** - el `<Footer />` ya está posicionado después de `<Routes>`
- El footer se renderizará naturalmente después del contenido de la ruta activa

## Cambio específico en Footer.jsx

**Antes:**
```jsx
<footer className="fixed bottom-0 left-0 right-0 z-50 bg-neutral-950 py-2 px-4">
```

**Después:**
```jsx
<footer className="bg-neutral-950 py-2 px-4">
```

## Verificación
1. Ejecutar `npm run dev` y navegar por todas las páginas
2. Verificar que el footer aparece después del contenido (no fijo en el viewport)
3. En páginas con poco contenido: el footer debe quedar inmediatamente debajo
4. En páginas con mucho contenido: el footer debe aparecer al final del scroll
5. Verificar que no hay contenido oculto detrás del footer
